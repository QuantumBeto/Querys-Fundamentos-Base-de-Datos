Q-01
autenticar(idUsuario, contrasena) Valida credenciales y retorna datos del usuario logueado
▶

Este query busca en la tabla USUARIOS un registro cuyo ID_USUARIO y CONTRASENA coincidan con lo ingresado en el login. Usamos UPPER() en el ID para que el login sea insensible a mayúsculas/minúsculas.

Si encuentra un registro, retorna un arreglo con: ID, NOMBRE, tipo de display ("Artista"/"Oyente") y el tipo raw ("Premium"/"Gratis"), que se usa en la lógica del Sidebar para mostrar u ocultar opciones de creación.

SELECT ID_USUARIO, NOMBRE, TIPODEUSUARIO
FROM   USUARIOS
WHERE  UPPER(ID_USUARIO) = UPPER(?)   -- insensible a mayúsculas
  AND  CONTRASENA       = ?           -- contraseña exacta

Q-02
generarNuevoId() Genera el siguiente ID secuencial (ART001, ART002…)
▶

Para no depender de una secuencia de Oracle, calculamos el próximo ID de forma manual. SUBSTR(ID_ARTISTA, 4) extrae los dígitos del ID (ej. "001" de "ART001"), TO_NUMBER los convierte a entero, MAX toma el mayor, y NVL(..., 0) maneja el caso de tabla vacía. En Java se formatea como String.format("ART%03d", ...).

SELECT NVL(MAX(TO_NUMBER(SUBSTR(ID_ARTISTA, 4))), 0) + 1
       AS NEXT_ID
FROM ARTISTAS
SELECT MAX / NVL

Q-04
obtenerTodos() Lista todos los artistas con su conteo de canciones
▶

Hace un LEFT JOIN con la tabla intermedia ARTISTAS_CANCIONES para contar cuántas canciones tiene cada artista. Se usa LEFT JOIN (no INNER) para que los artistas sin canciones también aparezcan. El GROUP BY agrupa por todos los campos no-agregados del artista.

SELECT A.ID_ARTISTA, A.NOMBRE, A.DESCRIPCION,
       A.GENEROPRINCIPAL, A.PAISDEORIGEN,
       COUNT(AC.ID_CANCION) AS TOTAL_CANCIONES
FROM      ARTISTAS A
LEFT JOIN ARTISTAS_CANCIONES AC ON A.ID_ARTISTA = AC.ID_ARTISTA
GROUP BY  A.ID_ARTISTA, A.NOMBRE, A.DESCRIPCION,
          A.GENEROPRINCIPAL, A.PAISDEORIGEN
ORDER BY  UPPER(A.NOMBRE)
SELECT LEFT JOIN GROUP BY / COUNT

Q-07
insertar(…) — Crear álbum Inserta un nuevo álbum con EMPTY_BLOB() para la portada
▶

El campo PORTADA es un BLOB de Oracle. La estrategia de dos pasos es obligatoria en Oracle JDBC antiguo: primero se inserta con EMPTY_BLOB() (un localizador vacío) y luego se escribe el contenido binario. Aquí solo se hace el primer paso; la portada se llena desde otro flujo.

El ID se genera con el mismo patrón NVL+MAX que Q-02, produciendo ALB001, ALB002, etc.

INSERT INTO ALBUMES
  (ID_ALBUM, TITULO, PORTADA, NUMERODECANCIONES,
   GENERO, FECHA, DURACION_SEG, COMPOSITORES, ID_ARTISTA)
VALUES
  (?, ?, EMPTY_BLOB(), ?, ?, ?, ?, ?, ?)
INSERT BLOB

Q-17
obtenerHistorialUsuario(idUsuario) Historial deduplicado con funciones de ventana
▶

Este es el query más avanzado del proyecto. Usa dos funciones de ventana de Oracle:

• MAX(...) OVER (PARTITION BY C.ID_CANCION): calcula la última vez que se escuchó cada canción, sin perder las otras filas.

• ROW_NUMBER() OVER (PARTITION BY C.ID_CANCION ORDER BY HC.FECHA_REPRODUCCION DESC): numera las reproducciones de cada canción de más reciente a más antigua.

La subconsulta exterior filtra solo WHERE RN = 1, quedándonos con la reproducción más reciente de cada canción sin duplicados. Luego ordena por ULTIMA_VEZ DESC.

SELECT * FROM (
  SELECT C.ID_CANCION, C.NOMBRE, C.GENERO,
         A.NOMBRE AS ARTISTA, C.FT, C.PORTADA,
         C.EMOCION, C.DURACION_SEG, C.FECHA,
         MAX(HC.FECHA_REPRODUCCION)
           OVER (PARTITION BY C.ID_CANCION)    AS ULTIMA_VEZ,
         ROW_NUMBER()
           OVER (PARTITION BY C.ID_CANCION
                 ORDER BY HC.FECHA_REPRODUCCION DESC) AS RN
  FROM  CANCIONES          C
  JOIN  HISTORIAL_CANCIONES HC ON C.ID_CANCION   = HC.ID_CANCION
  JOIN  HISTORIAL           H  ON HC.ID_HISTORIAL = H.ID_HISTORIAL
  JOIN  ARTISTAS_CANCIONES  AC ON C.ID_CANCION   = AC.ID_CANCION
  JOIN  ARTISTAS            A  ON AC.ID_ARTISTA  = A.ID_ARTISTA
  WHERE H.ID_USUARIO = ?
)
WHERE RN = 1
ORDER BY ULTIMA_VEZ DESC

Q-28
obtenerGenerosMasEscuchadosConFecha(…) Géneros más escuchados con filtro de rango de fechas opcional
▶

Hay dos variantes: con y sin rango de fechas. Cuando se pasan fechas, se une con RESUMENES_SEMANALES y se filtra con BETWEEN. Agrupa por género y ordena por total descendente para mostrar el más escuchado primero.

-- Con filtro de fechas:
SELECT C.GENERO, COUNT(C.ID_CANCION) AS TOTAL
FROM  CANCIONES           C
JOIN  HISTORIAL_CANCIONES HC ON C.ID_CANCION   = HC.ID_CANCION

JOIN  RESUMENES_SEMANALES RS ON RS.ID_HISTORIAL = H.ID_HISTORIAL
WHERE H.ID_USUARIO = ?
  AND  RS.FECHA BETWEEN ? AND ?
GROUP BY C.GENERO
ORDER BY TOTAL DESC
SELECT GROUP BY JOIN ×4 / BETWEEN
Q-29
obtenerCancionesEnRango(idUsuario, desde, hasta) Canciones únicas escuchadas en un período — subconsulta con DISTINCT
▶

Usa una subconsulta para obtener canciones únicas con DISTINCT, y luego hace un JOIN adicional con CANCIONES (alias C2) para obtener la portada (BLOB). La portada no se incluye en el DISTINCT para no complicar la deduplicación.

SELECT sub.*, C2.PORTADA
FROM (
  SELECT DISTINCT C.ID_CANCION, C.NOMBRE, ...
  FROM  CANCIONES C
  JOIN  ... -- cadena de JOINs hasta RESUMENES_SEMANALES
  WHERE H.ID_USUARIO = ?
    AND  RS.FECHA BETWEEN ? AND ?
) sub
JOIN CANCIONES C2 ON C2.ID_CANCION = sub.ID_CANCION
ORDER BY UPPER(sub.NOMBRE)

Q-30
obtenerHistorialGeneral() Historial global deduplicado con ROW_NUMBER()
▶

Similar a Q-17 pero sin filtrar por usuario. Usa ROW_NUMBER() OVER (PARTITION BY C.ID_CANCION) para numerar duplicados de cada canción y luego filtra RN = 1 para obtener una sola aparición por canción.

  JOIN ARTISTAS A ...
)
WHERE RN = 1
